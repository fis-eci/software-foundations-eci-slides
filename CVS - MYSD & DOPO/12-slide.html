S3: COMANDOS BÁSICOS (2/2)

Flujo Diario y Ramas

1. Preparar Cambios Paso 1

Consulta archivos modificados y agrégalos al "staging" antes de guardarlos.

# Ver estado (archivos rojos = sin preparar) $ git status # Preparar TODOS los cambios (pasan a verde) $ git add .
. Todo el directorio actual
2. Guardar Versión Paso 2

Empaqueta lo preparado en una nueva versión. Usa mensajes descriptivos.

# Guardar con mensaje explicativo $ git commit -m "feat: crear clase Estudiante"
-m Mensaje
feat: Funcionalidad
fix: Corrección
3. Gestión de Ramas Aislamiento

Crea una línea paralela para trabajar sin dañar el código principal.

# Crear rama y moverse a ella inmediatamente $ git checkout -b feature/sanchez # Listar ramas (asterisco * indica la actual) $ git branch
-b Crear + Mover
feature/ Prefijo recomendado
4. Sincronización Nube

Sube tu trabajo a GitHub o descarga lo nuevo frecuentemente.

# Subir tu rama a la nube (GitHub) $ git push origin feature/sanchez # Bajar cambios del equipo (actualizar local) $ git pull origin main
origin Remoto
main Rama estable
