S1: TEORÍA

Acciones Locales ⇄ Remotas

Local → Remoto
Publicar (Push)

Envía tus versiones y ramas al servidor. Hace visible tu trabajo para los demás.

git push origin rama
Local → Remoto
Proponer (Pull Request)

Solicitas que tus cambios se unan al tronco principal. Permite la revisión por pares.

Interfaz Web (GitHub)
Remoto → Local
Transmitir (Fetch/Pull)

Descarga la evolución oficial del proyecto. Actualiza tu entorno con el trabajo de otros.

git pull origin main
En Remoto
Combinar (Merge)

Une las contribuciones aprobadas al proyecto unificado. Cierra el ciclo de colaboración.

Botón "Merge Pull Request"
Remoto → Local
Consultar (Fetch)

Solo verifica si hay cambios nuevos en el servidor sin descargarlos ni mezclarlos aún.

git fetch origin
Remoto → Local
Clonar (Clone)

Descarga una copia completa del repositorio remoto por primera vez a tu máquina.

git clone url-repositorio

Conflictos de Fusión (Merge Conflicts)

Si Git encuentra cambios contradictorios al traer actualizaciones, marcará el archivo con:

<<<<<<< HEAD (Mis cambios) vs ======= (Los otros) >>>>>>>
