S1: TEORÍA

Acciones Básicas en el Repositorio Local

Crear Repositorio

Establece el historial inicial. Transforma una carpeta común en un proyecto con control de versiones.

git init
Crear Versiones (Commit)

Guarda una foto exacta del proyecto en ese instante. Agrega un nuevo vértice al historial.

git commit -m "mensaje"
Crear Ramas

Abre una línea de desarrollo paralela para trabajar sin afectar el código principal.

git branch nombre-rama
Ubicar Versión (Checkout)

Te mueve a través del historial. Puedes saltar entre ramas o volver a versiones pasadas.

git checkout nombre-rama
Fusionar Líneas (Merge)

Combina dos historias paralelas. Une los cambios de una rama secundaria hacia la principal.

git merge rama-a-unir
Revertir Estado

Deshace cambios o elimina versiones erróneas. Útil para corregir errores graves.

git reset / git revert

Manejo de Conflictos

Ocurre cuando dos personas modifican la misma línea del mismo archivo de forma diferente. El sistema no sabe con cuál quedarse y pide ayuda manual.
