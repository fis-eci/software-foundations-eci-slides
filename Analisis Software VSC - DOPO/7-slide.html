ANÁLISIS 4🔴 PIT con Maven — Pruebas de Mutación
Configurar pom.xml
<plugin>
  <groupId>org.pitest</groupId>
  <artifactId>pitest-maven</artifactId>
  <version>1.15.0</version>
  <configuration>
    <targetClasses>
      <param>com.tupackage.*</param>
    </targetClasses>
    <targetTests>
      <param>com.tupackage.*Test</param>
    </targetTests>
  </configuration>
</plugin>
Este bloque va dentro de <build><plugins>...</plugins></build>
Ejecución e interpretación
TERMINAL — bash
$ mvn pitest:mutationCoverage
[INFO] Scanning for projects...
[INFO] Generated 12 mutations
[INFO] 10 mutations killed (83%)
[INFO] BUILD SUCCESS
Killed
10/12 — pruebas eficaces
Survived
2/12 — pruebas débiles ← acción requerida
Survived:  monto > 0→monto >= 0  ❌ ninguna prueba falló
@Test
public void shouldNotConsignarWhenMontoCero() {
  cuenta.consignar(0.0);
  assertEquals(0.0, cuenta.getSaldo(), 0.01);
}
⏸️ Cuando veas un mutante Survived, pausa y escribe la prueba que lo mataría.
