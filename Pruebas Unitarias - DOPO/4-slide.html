Nomenclatura Estricta

Convention: shouldXWhenY

should ComportamientoEsperado When CondiciónOEstado

Caso Nominal
CalculatorTest.java
@Test
public void shouldCalculateTotalWhenItemsAreValid() {
  // Given (Dado)
  Cart cart = new Cart();
  cart.add(new Item(100));

  // When (Cuando)
  int total = cart.calculateTotal();

  // Then (Entonces)
  assertEquals(100, total);
}
Caso de Error
UserServiceTest.java
@Test
public void shouldThrowExceptionWhenUserIsNull() {
  // Given
  User user = null;

  // When / Then
  assertThrows(IllegalArgumentException.class, () -> {
    service.register(user);
  });
}

Regla de Oro de DOPO:

El nombre de la prueba es su mejor documentación. Si una prueba falla en el servidor, debes poder entender qué se rompió y bajo qué circunstancias solo leyendo el nombre del método, sin mirar el código.

DOPO - Desarrollo Orientado por Objetos
