Las 3 Reglas de Oro

1
Code the Unit Test First TDD

Antes de escribir código funcional, escribe la prueba.
Ciclo: Rojo (Falla) → Verde (Pasa) → Refactor (Mejora).

2
All Code Must Have Tests Zero Tolerance

No se entrega ni se hace commit de código sin pruebas asociadas.
La cobertura debe ser total para la lógica de negocio crítica.

3
Found a Bug? Create a Test! Regression Shield

Si reportan un error, primero crea una prueba que replique el fallo (rojo).
Solo entonces arregla el código. Esto evita que el error vuelva a ocurrir.

DOPO - Desarrollo Orientado por Objetos

Disciplina = Calidad de Software
