Fundamentos: Pruebas Unitarias

¿QUÉ ES?

  • Unidad Mínima: Evalúa el fragmento más pequeño de código (generalmente un método) de forma aislada.

  • Aislamiento Total: Sin dependencias externas reales (BD, Red, Sistema de archivos).

  • Foco Específico: Valida una única lógica o comportamiento concreto.

¿POR QUÉ?

  • Previene Regresiones: Detecta si cambios recientes rompieron código que ya funcionaba.

  • Documentación Viva: Los tests explican el comportamiento esperado mejor que un manual.

  • Confianza (Refactor): Permite mejorar la estructura interna sin miedo a romper la funcionalidad.
