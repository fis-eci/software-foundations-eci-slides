Tipos de Pruebas en DOPO

Caso Nominal

El "Camino Feliz" (Happy Path). Entradas válidas donde todo funciona según lo esperado y devuelve un resultado correcto.

Ejemplo

// Factorial de 5 es 120
factorial(5) → 120

Caso de Borde

Valores límite o extremos donde la lógica suele fallar. Puntos de inflexión (0, null, vacíos, máximos).

Ejemplo

// Factorial de 0 es 1
factorial(0) → 1

// Lista vacía size = 0

Caso de Error

Entradas inválidas que el sistema debe rechazar explícitamente, generalmente lanzando una Excepción.

Ejemplo

// No existe factorial negativo
factorial(-1)
throws IllegalArgumentEx

DOPO - Desarrollo Orientado por Objetos

Cobertura: Feliz + Borde + Error = Calidad
