CONTROL DE VERSIONES
Versionando tus Scripts .sql con Git
ESTRUCTURA RECOMENDADA DEL PROYECTO
mi-proyecto/
├── src/
│   └── *.java          ← código Java del proyecto
├── scripts/
│   ├── esquema.sql     ← DDL: CREATE TABLE, ALTER
│   ├── datos.sql       ← DML: INSERT, seed data
│   └── consultas.sql   ← SELECT y reportes
├── .gitignore          ← excluye .class, target/
└── README.md          ← descripción del proyecto
.GITIGNORE — PLANTILLA RECOMENDADA
# Compilados Java
*.class  *.jar  target/
# Forzar rastreo de SQL
!*.sql           ← los scripts siempre se versionan
# Logs y temporales
*.log  .env  *.tmp
COMMITS CORRECTOS
git commit -m "feat: añade esquema inicial con claves foráneas"
git commit -m "fix: corrige restricción NOT NULL en tabla cliente"
git commit -m "feat: agrega datos de prueba para cliente y cuenta"
COMMITS INCORRECTOS
git commit -m "cambios sql"
git commit -m "listo"
git commit -m "correcciones"
Principio: Un commit = un cambio atómico con propósito claro y mensaje descriptivo.
FLUJO COMPLETO POR ENTREGA
git add scripts/esquema.sql      # solo el archivo relevante
git commit -m "feat: CREATE TABLE clientes"
git push origin main             # sube al repositorio
