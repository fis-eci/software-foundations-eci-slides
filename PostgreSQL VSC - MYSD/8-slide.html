CONTROL DE VERSIONES
Versionando tus Scripts .sql con Git
ESTRUCTURA DEL PROYECTO
mi-proyecto/
├── src/
│   └── *.java          ← código Java
├── scripts/
│   ├── esquema.sql     ← DDL: CREATE TABLE
│   └── datos.sql       ← DML: INSERT, SELECT
└── .gitignore          ← incluye !*.sql
El .gitignore estándar del curso ya incluye !*.sql — tus scripts quedan rastreados automáticamente.
COMMITS CORRECTOS
git commit -m "feat: añade esquema inicial con claves foráneas"
git commit -m "fix: corrige restricción NOT NULL en tabla cliente"
git commit -m "feat: agrega datos de prueba para cliente y cuenta"
COMMITS INCORRECTOS
git commit -m "cambios sql"
git commit -m "listo"
git commit -m "correcciones"
Principio: Un commit = un cambio atómico con propósito claro.
