ANÁLISIS 2 🟠 PMD — Análisis de Calidad y Malas Prácticas

Procedimiento de Ejecución

1 Clic derecho sobre tu proyecto
2 PMD → Check Code
3 Ordena vista PMD Violations por "P"

Tabla de Prioridades en PMD

Prioridad 1 Crítico (posible bug real en runtime) OBLIGATORIO
Prioridad 2 Alta (problema grave de calidad) JUSTIFICAR
Prioridad 3-5 Media/Baja (estilo o menor) OPCIONAL
💡 Consejo: Enfócate siempre en resolver primero los incidentes de prioridad 1 y 2. Las prioridades 3, 4 y 5 corresponden a formato de código.

Caso de Estudio: Método Inútil

PMD WARNING: UnusedPrivateMethod
private void metodoInterno() {  // Unused
    System.out.println("debug");
}
Diagnóstico técnico: Método privado que nunca se invoca dentro de la clase. Representa código muerto, viola el principio ágil YAGNI (You Aren't Gonna Need It).
Solución recomendada: Eliminar el método inactivo por completo para mantener el proyecto limpio.
⏸️
ACTIVIDAD DE PAUSA: Antes de observar la solución, escribe en un papel tu propia propuesta detallada de corrección para el método privado no utilizado.
