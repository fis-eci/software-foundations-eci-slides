S4 PROTOCOLO Ciclo de Análisis: Las 5 Etapas para cada Herramienta
I
Análisis Inicial
Ejecuta la herramienta y registra TODOS los incidentes en una tabla.
II
Interpretación
Lee cada incidente de software. Clasifícalo de acuerdo con la prioridad.
III
Selección
Elige el incidente más relevante. Justifica técnicamente la selección.
IV
Solución
Aplica la corrección en el código Java y documenta el razonamiento.
V
Análisis Final
Vuelve a ejecutar la herramienta y compara: Estado Inicial vs Final.
Orden de ejecución: aplica este ciclo de cinco etapas secuencialmente para SonarLint → PMD → JaCoCo → Pitclipse. Cada herramienta produce su propio reporte independiente.

Entregable Esperado

Para cada una de las cuatro herramientas de análisis deberás consolidar e incluir en tu reporte final:

  • Tabla Inicial: Listado y conteo de todos los incidentes detectados antes de realizar cambios.
  • Justificación de Selección: ¿Por qué seleccionaste ese incidente específico para resolver?
  • Código Antes/Después: Evidencia del código fuente previo y el corregido.
  • Tabla Final: Reporte que evidencia la reducción o eliminación de fallos.
📋 Repite este ciclo de forma rigurosa para las cuatro herramientas antes de la fecha de entrega.
